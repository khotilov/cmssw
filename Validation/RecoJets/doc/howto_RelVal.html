How to do a validation?

1) First check out the latest version of Jet Plus Tracks. The instructions can be found under: https://twiki.cern.ch/twiki/bin/view/CMS/JetPlusTracksCorrections

2) cvs co Validation/RecoJets
  Check out the JetMet release validation package.

3) cmsRun alljets_validation_cfg.py
  Edit the alljets_validation_cfg.py file, and define over which samples you would like to run. For comparing two different releases you have to run alljets_validation_cfg.py two times (once for each sample).
  You get two root files containing the histograms.

4) ./validation.sh
  Run the validation.sh script. This script produces all the plots that are available. You are asked by the script to enter the name of the files you want to compare and the name of the directory where the plots should be stored. This name is also the title in the histograms.

5) Open the index.html file (located in the new directory) in a web browser. Now you can look at all the plots...

Example:

The alljets_validation_cfg.py runs per default over a CMSSW_2_1_10/RelValQCD_Pt_80_120/GEN-SIM-DIGI-RAW-HLTDEBUG-RECO/STARTUP_V7_v2 sample.
  cmsRun alljets_validation_cfg.py
The histogramms are now saved in a file called QCD_80_120_STARTUP_V7_v2.root
To get the plots run now the script:
  ./validation
The prompt shows:
  This script produces all the histograms and plots...
  Enter the REFERNCE.root file, the NEW.root file and the directory name/histo title([REFERNCE.root] [NEW.root] [title]):
For a test compare the file to itself, and safe the files in a directory called testdir
  QCD_80_120_STARTUP_V7_v2.root QCD_80_120_STARTUP_V7_v2.root testdir
now look at the plots...
  firefox testdir/index.html
